🔴 Issues Found in Current System

#IssueImpactSeverityFix
1 Duplicate Action Tables
Both actions and audit_actions exist. Actions from hazards use one table; actions from audits use a separate table. They are never seen together.		 Action dashboard is split. Can't see ALL open actions in one view. Critical Merge AuditAction into Action using source field
2 Duplicate Audit/Finding Tables
Legacy Audit + Finding models exist alongside the new AuditPlan + AuditSchedule + AuditFinding system. Routes at /audits/* use the old models.		 Two audit systems running in parallel. Data is split. Confusion for users. Critical Remove /audits/* old routes and redirect to /audit-schedule
3 MOC Not Generating Actions
MOC creates a Hazard but does not create an Action in the unified action system. Post-change actions have no tracker.		 MOC changes are not followed up. No accountability. High Add Action creation in MOC submit route using unified Action model
4 Investigation Actions Not in Unified System
Investigation recommendations generate an Action but the link is stored in action.linked_ref_id only. Not visible in the main action dashboard filter.		 Investigation follow-up actions are hidden from the main Actions view. High Ensure source='Investigation' is set so filter works correctly
5 Safety Promotion Under Wrong Module
Bulletins and training records are in a separate "Safety Promotion" nav section disconnected from Module 1 (Safety Policy).		 ICAO Module 4 (Safety Promotion) is not clearly separated from Module 1 in navigation. Medium Rename nav section. Move Training Records under Safety Promotion clearly.
6 Navigation Not Module-Based
Sidebar lists individual pages rather than ICAO modules. A new user can't understand what belongs to what.		 Poor usability. Not aligned with ICAO/IOSA module structure. Medium Restructure sidebar into 4 clear ICAO pillars with sub-items
7 Dashboard Not Unified
Main dashboard shows hazards and actions separately. Audit dashboard is a separate page. There is no single SMS health view.		 Management cannot see the full SMS status in one view. Medium Upgrade main dashboard to show all 10 modules with status indicators

🔗 Core Entity Relationships

Every entity must be connected. No entity should be isolated.

Primary Relationships

Report → creates Hazard
Hazard → has multiple Risks
Risk → generates Actions
Audit Finding → creates Actions
Investigation → creates Actions
MOC → creates Risk → creates Actions
Document → linked to ALL modules

Action Sources (unified)

All actions must go into ONE table with source field:
source = 'Hazard Report'✓
source = 'ASR'✓
source = 'Audit'⚠ Currently in AuditAction
source = 'Investigation'✓ (partial)
source = 'MOC'✗ Missing

🛠 Cleanup Action Plan — Priority Order

1

Fix: Unify Action Tables Critical · Do First

Route /audit-actions to display records from the main Action table filtered by source='Audit'. When creating audit actions (in new_finding()), write to Action model, not AuditAction. This gives one unified view. Mark AuditAction as deprecated.

2

Fix: Remove Legacy Audit Routes Critical

Routes /audits, /audits/new, /audits/<id>, /audits/<id>/add-finding, /audits/<id>/update use the legacy Audit + Finding models. Remove these routes and add a redirect from /audits to /audit-schedule. The new system is complete and replaces them.

3

Fix: MOC Must Generate Action High

In the new_moc() route, after creating the hazard, also create an Action record with source='MOC', linked to the MOC ID. This ensures MOC changes have tracked follow-up actions visible in the unified action dashboard.

4

Fix: Restructure Navigation into 4 Pillars High

Update base.html sidebar to use the 4 ICAO pillars structure shown above. Safety Promotion must be its own pillar. Risk Matrix moves to Pillar 4. All audit sub-pages move under Pillar 3.

5

Fix: Upgrade Main Dashboard Medium

Main dashboard at / should show all 10 SMS module statuses. Add: total audit actions (not separate from main actions), SPI alert count, active documents count, and a module health grid with green/yellow/red indicators per pillar.

6

Fix: Investigation Action Visibility Medium

Confirm source='Investigation' is set correctly on actions created from investigations. Add "Investigation" as a filter option in the /actions page filter bar. Verify the link in action dashboard shows the investigation ID as a clickable reference.

7

Future: Remove Deprecated Models Low · After Migration

Once all data is confirmed migrated to the new system, remove class Audit, class Finding, and class AuditAction from models.py. Drop the corresponding database tables. This reduces confusion and keeps the model layer clean.
